Frequently Asked Questions

What is timeboxing?

Timeboxing assigns a fixed block of time to a task and stops when that time is up — regardless of whether the task feels "done." It fights perfectionism, prevents overrun, and forces realistic scheduling. It's one of the most effective time management methods for deep work and ADHD-friendly productivity.

How does a Box work?

  1. Brain Dump — add every task you want to do today, no order needed.
  2. Estimate — set how long each task actually needs (be honest).
  3. Arrange — put tasks in the order you'll do them. BoxIt shows projected finish times so you can see if the plan is realistic before committing.
  4. BoxIt! — commit the plan. The Box is locked.
  5. Start — tap Start on any block. A timer runs. When it ends, a hard-stop alarm fires.
  6. Review — see how your estimates compared to reality. Improve over time.

Why does the alarm ring even when my phone is on silent?

This is intentional. BoxIt uses AlarmKit — the same system powering iOS native alarms — to ring a hard stop when a time block ends. A timebox that respects silent mode isn't a timebox. You can disable AlarmKit in Settings → Notifications → BoxIt, but this removes the core hard-stop behavior.

Can I start tasks in a different order?

Yes. On the Day tab, tap Start on any queued block in any order. Projected finish times update in real time. The order of the Box is a plan, not a lock.

Can I edit tasks after committing a Box?

No. Once you tap "BoxIt!", the Box is committed. If you need to change tasks, start a new Box. This is by design — the commitment is the point.

Where is my data stored?

Everything is stored locally on your device. Nothing leaves your phone. There is no account, no cloud sync, and no backup. If you delete the app, your data is permanently gone.

Does BoxIt work on iPad?

BoxIt is an iPhone app. It will run on iPad in iPhone compatibility mode but is not optimized for iPad layouts.

I found a bug — how do I report it?

Open an issue at github.com/ryanvreeke/BoxIt/issues or email ryanvreekedev@gmail.com, or use the feedback form. Include your iOS version, steps to reproduce, and screenshots if relevant.
